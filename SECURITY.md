Reporting a Vulnerability

We take security vulnerabilities seriously and appreciate responsible disclosure.

Please do not report security issues via public GitHub issues, discussions, or social media.

Instead, report vulnerabilities using one of the following private channels:

Email: bloomeor@gmail.com

Subject: Security Vulnerability Report

When reporting, please include:

A clear description of the vulnerability

Steps to reproduce the issue

Affected versions or components

Proof of concept (if available)

Your contact information (optional but helpful)

What to Expect

Acknowledgement: Within 48 hours

Initial Assessment: Within 5 business days

Fix & Disclosure: Timeline will depend on severity and complexity

We aim to work collaboratively with reporters to validate and remediate issues as quickly as possible.

Responsible Disclosure

We kindly ask that you:

Allow us reasonable time to investigate and fix the issue before public disclosure

Avoid accessing, modifying, or deleting user data

Avoid actions that could degrade service availability

Security Best Practices

If you are deploying or using this project:

Keep dependencies up to date

Use HTTPS and secure headers

Follow the principle of least privilege

Regularly audit logs and access controls

Credits

We appreciate and acknowledge security researchers and community members who responsibly disclose vulnerabilities.
